# Array（陣列）

## 1. Array 是什麼

- Array 是一種將資料存放在「連續記憶體空間」中的資料結構
- 每個元素都有固定的位置（index，通常從 0 開始）
- 可以使用 `array[i]` 直接存取指定元素

### 適合用途

- 儲存大量同類型資料
- 需要頻繁讀取資料、但較少修改資料的情境

### 核心特性

- 記憶體連續配置
- 支援 Random Access
- 存取時間複雜度為 `O(1)`

---

## 2. 一維陣列（1D Array）

- 元素以直線方式排列
- 每個元素只需要一個 index 存取

### 常見用途

- 成績列表
- 分數、數字清單
- 資料序列

### 特性

- index 連續
- 記憶體中連續存放

---

## 3. 二維陣列（2D Array）

- 表格型結構
- 由列（row）與欄（column）組成
- 常見存取方式為 `array[row][col]`

### 常見用途

- 成績表
- 棋盤
- 矩陣運算

---

## 4. 三維陣列（3D Array）

- 多個二維陣列堆疊而成
- 多一個「層（layer）」的概念
- 常以「層 / 列 / 欄」存取

### 常見用途

- 影像資料（高、寬、顏色）
- 影片資料（時間序列）
- 空間或立體資料

---

## 5. Static Array（靜態陣列）

- 陣列大小在宣告時就決定
- 執行期間無法改變大小

### 特性

- 存取速度快
- 結構簡單
- 無法動態擴充
- 容易發生 overflow

### 適合情境

- 資料量一開始就能確定
- 結構固定、不需擴充

---

## 6. Dynamic Array（動態陣列）

- 使用 `malloc` / `realloc` 配置記憶體
- 陣列大小可在執行期間調整

### 特性

- `realloc` 可能更換記憶體位址
- 必須手動 `free`
- 若管理不當容易造成 memory leak

### 適合情境

- 資料數量不確定
- 會隨時間增減

---

## 7. Access（存取）

- 使用 index 直接存取指定位置的資料
- 不需走訪其他元素

### 時間複雜度

- `O(1)`

---

## 8. Insert / Delete（插入 / 刪除）

- 在中間插入或刪除資料時
- 後方元素必須整體向前或向後移動

### 特性

- 記憶體需維持連續
- 平均時間複雜度為 `O(n)`

---

## 9. Traverse（走訪）

- 從第一個元素依序走訪到最後一個
- 常用於印出資料、計算統計值

### 時間複雜度

- `O(n)`

---

## 10. Linear Search（線性搜尋）

- 從頭開始逐一比較
- 不需要事先排序

### 特性

- 結構簡單
- 最壞情況需檢查所有元素
- 時間複雜度為 `O(n)`

---

## 11. Binary Search（二分搜尋）

- 每次將搜尋範圍切成一半
- 僅適用於「已排序」的陣列

### 特性

- 搜尋效率高
- 時間複雜度為 `O(log n)`

---

## 12. 排序演算法（Array 常見）

### Bubble Sort

- 相鄰元素兩兩比較
- 較大的值逐步往後移動
- 教學用途居多

### Selection Sort

- 每一輪選出最小元素
- 交換次數少
- 時間複雜度固定為 `O(n²)`

### Insertion Sort

- 將元素插入已排序區段
- 對「幾乎已排序」資料效率佳
- 最好情況為 `O(n)`

---

## 13. Array 的優缺點

### 優點

- 隨機存取快（`O(1)`）
- 記憶體連續，Cache 友善
- 結構簡單

### 缺點

- 插入與刪除成本高（`O(n)`）
- Static Array 大小固定
- Dynamic Array 需要額外記憶體管理

---

## Appendix A：Array 維度比較

| 維度 | 結構 | Index 數量 | 常見用途 |
|---|---|---|---|
| 1D | 一條直線 | 1 | 清單、分數 |
| 2D | 表格 | 2 | 成績表、棋盤 |
| 3D | 多層表格 | 3 | 影像、影片 |

---

## Appendix B：Static vs Dynamic Array

| 項目 | Static Array | Dynamic Array |
|---|---|---|
| 大小是否可變 | 否 | 是 |
| 配置時機 | 編譯期 | 執行期 |
| 記憶體管理 | 自動 | 手動 |
| 使用風險 | Overflow | Memory leak |

---

## Appendix C：時間複雜度整理

| 操作 | 時間複雜度 |
|---|---|
| Access | O(1) |
| Insert | O(n) |
| Delete | O(n) |
| Traverse | O(n) |
| Linear Search | O(n) |
| Binary Search | O(log n) |

---

## Appendix D：什麼時候該用 Array

| 情境 | 適合與否 | 原因 |
|---|---|---|
| 需要快速存取 | 適合 | O(1) |
| 資料量固定 | 適合 | Static Array |
| 常插入 / 刪除 | 不適合 | O(n) |
| 記憶體連續性重要 | 適合 | Cache 友善 |
| 資料量高度變動 | 不適合 | realloc 成本高 |
