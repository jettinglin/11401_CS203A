# Assignment V – Tree Data Structures

本專案為 **Data Structures（CSE CS203A）Assignment V**，  
主題為各種樹狀資料結構（Tree Data Structures）的定義、階層關係、建構方式與實際應用。

本作業透過概念整理與視覺化工具，理解不同樹狀結構在「結構限制」、「平衡策略」與「應用情境」上的差異。

---

## 作業目標（Objectives）

本作業的主要目標包含：

- 理解並區分各種樹狀資料結構的定義與特性  
- 建立樹狀結構之間的階層與轉換關係  
- 使用相同的一組整數資料，建構不同類型的樹  
- 說明各種樹狀結構在實際系統或演算法中的應用  
- 練習將 AI 作為學習輔助工具，而非直接產出作業內容  

---

## 專案內容概覽

本 README 對應作業內容，依照課堂作業說明分為以下六個部分：

### Section A：樹狀結構定義（Definitions）
說明以下樹狀資料結構的基本定義與結構限制：

- General Tree  
- Binary Tree  
- Complete Binary Tree  
- Binary Search Tree (BST)  
- AVL Tree  
- Red-Black Tree  
- Max Heap  
- Min Heap  

---

### Section B：樹狀結構階層與轉換（Hierarchy & Transformation）
整理各種樹狀結構之間的關係，說明從一般樹開始，  
在逐步加入結構、排序或平衡限制後，如何發展出不同的樹狀結構分支。

---

### Section C：樹狀結構建構（Tree Construction）
使用同一組固定整數資料，分別建構：

- Binary Tree  
- Complete Binary Tree  
- Binary Search Tree  
- AVL Tree  
- Red-Black Tree  
- Max Heap  
- Min Heap  

每一種樹皆搭配：
- 建構原則說明  
- 使用之視覺化工具  
- 建構結果截圖  

---

### Section D：實際應用情境（Applications）
針對每一種樹狀結構，舉出一個實際應用或演算法情境，  
並說明該樹狀結構為何適合該用途。

---

### Section E：結構比較與反思（Reflection）
比較 BST、AVL Tree 與 Red-Black Tree 在不同使用情境下的取捨，包括：

- 以搜尋為主的情境  
- 插入與刪除頻繁的情境  
- 靜態資料、僅需搜尋的情境
